Draw-while-talk (still the money shot)

Three concurrent channels: (a) audio out, (b) async tool → RPC → animate, (c) transcript. RPC ack returns when the animation starts, not when it ends — so speech never waits on SVG. Model speaks semantic targets (vertex); client owns geometry.

Three product layers

  • Board — MathCanvas owns pointers; new world-space annotation layer inside .mc-world
  • Presence — LumenOverlay (orb + transcript + mic); never steals board input
  • Transport — token server + Python agent + LiveKit room

Staying under 65k TPM plan/11

Audio is the budget. Controllable text is a rounding error — so the strategy is short turns + Mistral holds memory, not “trim the prompt more.”

Fig 5 · Four levers keep Live cheap
Lever 1
PULL board context
Model calls get_board_state only when about to draw. Free tier: push disabled. ~60 tokens on demand beats N pushes/min.
Lever 2
Rolling session summary
Mistral refreshes ≤2 sentences on events (wrong answer, step jump). Sent on lumen.board as summary field. Costs Mistral budget, not Gemini TPM.
Lever 3
Lean prompt + short turns
~150 token system prompt. Persona: 1–3 sentences then pause. Long monologues are the biggest leak.
Lever 4
Tiny draw payloads
Compact {op, args}. Concept anims advance by scene index: play_concept_scene(target, 2) — not a full re-description.
PressureMitigation
Audio-heavy minutesShort-turn rule; soft ~10 min session cap

Closing line from 11: Mistral is the always-on brain and memory; Gemini Live is a cheap, short-spoken mouth + hand that stays under 65k TPM.

Hard rules (still true)

  • Never regenerate an open lesson under the learner mid-Live session.

How to build it (combined order)

Each slice demos on its own. Suggested if you implement both plans.

  1. Live Phase 1 — voice overlay Orb + talk on today’s static scripts. Prove no board takeover.
  2. Gen Phase A — Module 1 with Mistral Streaming roadmap + Module 1 into MathCanvas.
  3. Live Phase 2 — draw-while-talk World-space annotations; needs good enriched diagrams.
  4. Gen Phase B–C — background modules + Gen UI rail SSE statuses, concurrency 2, tool widgets.
  5. Gen Phase D — Tavily curation Live web research → CuratedBrief → lessons grounded in sources.
  6. Gen Phase E — concept animations Primitive library + player + showConceptAnimation + play_concept_scene.
  7. Phase F — budget polish session-summary bridge, pull-only board context, cascade/OpenAI fallbacks.
Dev ritual (Live): three terminals — token server (:8787), Python agent worker, npm run dev. Content APIs run inside the TanStack Start app with server-side Mistral + Tavily keys (never VITE_-prefixed).

Success when both sides work

  • Onboard → streaming roadmap → Module 1 ready in ~10–20s with real math
  • Modules 2–3 flip to ready while you’re inside Module 1
  • Live: ask about the graph → talk + circle vertex + draw axis; marks survive zoom
  • Concept anim: ≥2 different primitives in a module; voice can advance scenes
  • Live stays under free-tier TPM via short turns + Mistral summary (not by starving grounding)
